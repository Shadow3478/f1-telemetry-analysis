beneath the stopwatch

// alignment_engine.py

Distance-Grid Alignment

Telemetry from two drivers can't be overlaid by timestamp. The engine interpolates all channels onto 1,000 equally-spaced distance samples, enabling true spatial comparison regardless of lap start position or fuel load.

scipy.interp1d
// delta_calculator.py

Cumulative Delta Time

Reconstructs time from velocity at each distance point: Δt = Σ(ds/v). The resulting 1,000-point array reveals exactly where on the lap a gap opens, peaks, and closes — not just the final margin.

numpy vectorised
// detector_pipeline.py

7 Insight Detectors

Braking point, trail braking, corner exit speed, throttle application, traction zone ramp rate, DRS efficiency, and sector gap detectors each score and quantify their contribution to the total lap delta.

heuristic + statistical
// narrative_gen.py

Dual-Audience Narratives

Every insight produces two outputs simultaneously: a casual narrative for fans ("braked 8m later into Turn 4") and an analyst breakdown with raw channel values, confidence score, and statistical consistency.

casual + analyst mode
// ensemble_align.py

Ensemble Lap Selection

Selects 3–5 representative laps per driver filtered by tire age, track condition, and time window. Insights are reported only when consistent across ≥60% of the representative set. Confidence is proportional to consistency rate.

quality-gated
// storage.py

Parquet + DuckDB Storage

All telemetry stored as columnar Parquet files queryable via DuckDB SQL — no database server required. A full race weekend fits in under 200MB. Pre-computed insight bundles via Celery reduce cold-query latency to under 200ms.

zero-server architecture
